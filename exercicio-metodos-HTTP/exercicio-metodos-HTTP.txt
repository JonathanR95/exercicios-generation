Exercícios sobre Métodos HTTP

1) O que são métodos HTTP?

Métodos HTTP ou verbos são requisições que podemos fazer ao navegar pelo internet. Quando estamos navegando na web nosso navegador está enviando solicitações para um servidor que por sua vez nos envia respostas em formato de ações especificas de acordo com o que foi solicitado. 

2) Quais são os métodos HTTP mais utilizados atualmente?

GET - quando pedimos a representação de um arquivo para a nossa máquima.

POST - É utilizado quando queremos enviar uma informação para um site ou para um banco de dados.

PUT - Atualiza, ou substitui uma informação que vai ser guardada na web.

DELETE - Exclui uma arquivo ou informação.  

3) Quais são os 2 principais métodos do HTTP? Qual é a diferença entre eles?

GET e POST - Com eles conseguimos manipular as  informação 
em um site ou em um banco de dados. Funcionam como "pergunta e resposta" entre o cliente e o servidor.

4) O que são requisições HTTP? Quais os tipos?

Requisições são informações enviadas pelo usuário com fim de manipular as ações de uma aplicação web, podendo iniciar, interromper ou manipular as ações da aplicação.

5) Como funciona uma requisição?

A requisição é quando o cliente "pede" para inserir informações em um servidor ou banco de dados. Esse pedido contém os dados a serem inseridos, e a aplicação vai definir se o cliente tem autorização para inserir esses dados, se as informações estão no formato correto, entre outras verificações. Todos os parametros sendo atendidos a informação é registrada no servidor.

6) O que é o método GET?

É um método para envio de dados, geralmente de um formulário, para um servidor manipulá-lo.

7) Qual é o protocolo de internet para transferência segura, com uso de certificado digital, utilizado em sites de compras eletrônicas?

HTTPS - ele criptografa a comunicação entre o browser e o servidor, tornando mais segura a navegação.

8) Quais os principais métodos de uma requisição REST?

As requisições são feitas através dos métodos: GET, PUT, POST, PATCH, DELETE.

9) Qual a diferença entre IPv4 e IPv6?

Um dos grandes Problemas do IPv4 era a sua escalabilidade, pois só compartaria 4 bilhões de endereços de IP em todo mundo, pois os endereços são compostos por 32 bits, e considerando que temos muitos endereços de IP reservados e muitos inativos, esse número poderia ser alcançado com o avanço da internet. Já o IPv6 utiliza 128 bits para compor o endereço e permite 340 undecilhões de endereços o que termina com esse problema de escalabilidade.

10) Faça um resumo sobre os seguintes protocolos: http, tcp, ip, udp, smtp e ftp.

HTTP: é o protocolo que permite que o nosso browser "leia" os arquivos html e outros arquivos que podemos acessar através da internet.

TCP: Protocolo de Controle de transmissão, ele é resposável por analisar os dados que chegam ao computador, coloca-los na ordem e verificar se não ouve falhas na comunicação.

IP: Internet Protocol é o protocolo que identifica as maquinas, tanto de clientes como de servidores, é uma espécie de CPF do computador.

UDP: é um protocolo de internet mais simples e sem conexão, no qual os serviços de verificação e recuperação de erros não são necessários.

SMTP: é um protocolo responsável pelo envio e recebimento de e-mails.

FTP: É um protocolo específico para envio de arquivos pela internet.

Como vimos existem vários protocolos para que possamos nos comunicar através da internet. Se não fossem esses protocolos a internet correria o risco de ser monopolizada por um empresa, ou ao menos não ser tão difundida como é hoje por que dispositivis de marcas diferente provavelmente não conseguiriam se comunicar.
